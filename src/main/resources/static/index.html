Coding Rules

Use Case Diagrams

  1. Actors IDs has PascalCase
  2. Use Cases IDs has camelCase
  3. Comments are used to divide document into sections
  4. Actors section declare Actors
  5. Use Cases section declare useCases
  6. Relationships section declare Actor - Actor, Actor - useCase and useCase - useCase relationships
  7. Always try to place elements grouped and if possibile using all space available (left, up, right, down arrows)
